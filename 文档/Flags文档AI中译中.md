# 🚩 AppOpt (Aloazny二改) Flags 功能说明书

> **小白必读**：这些 `Flags` 就像是模块的“开关”。
> 你只需要在 `/data/adb/modules/AppOpt_Aloazny/Flags/` 目录下创建或删除对应的**空文件**
> 就能控制模块的功能。
> 
> **注意**：除了卸载，所有操作都不需要重启手机。
> 但修改 Flags 后，**必须重新刷入模块压缩包**才能让新设置生效！

---

### 📂 配置与更新 (决定谁说了算)

* **update_config** (核心开关)
    * **默认状态**：✅ 开启
    * **作用**：每次刷入都会用模块自带的最强配置覆盖你的旧文件。如果你想“放弃思考”直接用最稳的，选它。
* **keep_custom_rule** (增量更新)
    * **作用**：如果你自己手动改过规则，刷入时它会保留你的修改，并把开发者新加的规则补进去。不懂就去搜“增量更新”，总之就是你的规则优先。
* **zip_first** (包内优先)
    * **注意**：这个只能在**压缩包**里创建。
    * **作用**：强制让压缩包里的 Flags 覆盖手机里现有的 Flags。

### ⚙️ 核心适配 (机械化自动调校)

* **modtify_config** (自动化适配)
    * **默认状态**：✅ 开启 (对于 4+3+1 架构默认生效)
    * **作用**：根据你手机的 CPU 架构（如 `6+2`、`3+4+1`、`2+3+2+1`、`4+4`）自动修改核心配置，让 App 跑在最合适的核上。
* **dexota_modtify** (编译优化)
    * **作用**：修改 `dex2oat` 的编译参数。通俗点说，就是让手机在安装或后台编译 App 时，尽量多占用性能大核，提高运行效率。

### 🎮 游戏与兼容性 (清理冲突)

* **delete_game_config** (游戏一键清空)
    * **作用**：直接删掉模块自带的所有游戏配置。如果你觉得模块的游戏调度不舒服，用这个。
* **keep_Asoulopt** (Asoul 兼容模式)
    * **作用**：如果你同时装了 A-soul 模块，开启这个会精准删除与之冲突的游戏配置，保证两个模块不打架。
* **disable_program / enable_program** (系统进程屏蔽)
    * **作用**：控制是否干掉 `Joyose` 或 `oiface` 等系统调度。默认不做修改，除非你知道你在干什么。

---

### 🛠 操作速查表

| 操作类型 | 是否需要重启手机 | 关键步骤 |
| :--- | :--- | :--- |
| **安装/修改规则** | ❌ 不需要 | 建立/删除 Flag -> 重新刷入模块 |
| **生效/修改 Flags** | ❌ 不需要 | 建立/删除 Flag -> 重新刷入模块 |
| **卸载模块** | ✅ **需要** | Magisk/KSU 卸载 -> 重启手机 |

#### 📝 各个 Flags 大白话说明

| Flag 文件名 | 默认状态 | 一句话说明 (大白话) |
| :--- | :--- | :--- |
| **update_config** | ✅ 开启 | “无脑模式”，刷入就用模块自带的最新配置。 |
| **modtify_config** | ✅ 开启 | “自动排座”，让 App 自动找对 CPU 核心。 |
| **keep_custom_rule** | ❌ 关闭 | “我的优先”，更新时保留你偷改的规则。 |
| **dexota_modtify** | ❌ 关闭 | “编译加速”，让 App 编译时多用大核。 |
| **delete_game_config** | ❌ 关闭 | “不要游戏”，把模块里的游戏规则全删了。 |
| **keep_Asoulopt** | ❌ 关闭 | “和平共处”，防止跟 A-soul 模块打架。 |
| **disable_program** | ❌ 关闭 | “干掉系统”，屏蔽 Joyose 或 oiface 干扰。 |
| **zip_first** | ❌ 关闭 | “强行覆盖”，压缩包里的 Flag 说了算。 |

---
**💡 提示**：想要查看当前生效情况？刷入后看一眼 `module.prop` 的描述信息，✅ 代表开启，❎️ 代表关闭。
